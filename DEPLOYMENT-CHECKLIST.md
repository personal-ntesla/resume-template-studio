# GitHub Pages Deployment Checklist

1. Confirm the repository Settings → Pages source remains **GitHub Actions**.
2. Keep `public/CNAME` as `template.resumetemplate.cn`.
3. Run locally:

```bash
bun install
bun run build
bun run verify:pages
```

4. Commit all files, including the hidden `.github/workflows/deploy.yml` directory and the `bun.lock` generated by Bun:

```bash
git add -A
git commit -m "fix: deploy built Vite site to GitHub Pages"
git push origin main
```

5. In GitHub → Actions, wait for the **Deploy to GitHub Pages** workflow to finish with both jobs green.
6. Force-refresh the custom domain with Command + Shift + R.

## Expected network requests

A correct deployed build loads:

```text
/assets/index-<hash>.js
/assets/index-<hash>.css
/template-assets/<template-id>/page-1.jpg
```

It must **not** request:

```text
/src/main.tsx
/main.tsx
```
